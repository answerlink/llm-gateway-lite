# 日志轮转配置说明

## 📋 概述

项目配置了两层日志轮转机制：
1. **Docker 容器日志轮转**：控制 `docker logs` 输出的日志
2. **应用日志文件轮转**：控制 `logs/` 目录下的日志文件

---

## 1️⃣ Docker 容器日志轮转

### 配置位置
`docker-compose.yml`

### 当前配置

```yaml
services:
  llm-gateway:
    logging:
      driver: "json-file"
      options:
        max-size: "10m"      # 单个日志文件最大 10MB
        max-file: "3"        # 保留最近 3 个日志文件
        compress: "true"     # 压缩轮转的日志文件
```

### 参数说明

| 参数 | 值 | 说明 |
|------|------|------|
| `driver` | `json-file` | 使用 JSON 文件日志驱动（Docker 默认） |
| `max-size` | `10m` | 单个日志文件最大 10MB |
| `max-file` | `3` | 最多保留 3 个日志文件 |
| `compress` | `true` | 自动压缩轮转的日志文件 |

### 日志文件位置

Docker 容器日志存储在：
```bash
/var/lib/docker/containers/<container_id>/<container_id>-json.log
/var/lib/docker/containers/<container_id>/<container_id>-json.log.1.gz
/var/lib/docker/containers/<container_id>/<container_id>-json.log.2.gz
```

### 轮转效果

```
当前日志文件: container-id-json.log (9.5 MB)
                ↓ 继续写入，达到 10MB
                ↓
轮转发生:       container-id-json.log (新文件，从 0 开始)
                container-id-json.log.1.gz (压缩后的旧文件)
                ↓ 再次达到 10MB
                ↓
再次轮转:       container-id-json.log (新文件)
                container-id-json.log.1.gz (最新的旧文件)
                container-id-json.log.2.gz (更旧的文件)
                ↓ 再次达到 10MB
                ↓
继续轮转:       container-id-json.log (新文件)
                container-id-json.log.1.gz
                container-id-json.log.2.gz
                (最老的 .3.gz 被删除)
```

### 查看容器日志

```bash
# 查看实时日志
docker-compose logs -f llm-gateway

# 查看最近 100 行
docker-compose logs --tail=100 llm-gateway

# 查看带时间戳
docker-compose logs -t llm-gateway

# 查看日志文件大小
docker inspect llm-gateway-lite-llm-gateway-1 --format='{{.LogPath}}' | xargs ls -lh
```

### 调整配置

根据实际需求可以调整参数：

**开发环境**（日志量小，保留时间短）:
```yaml
logging:
  driver: "json-file"
  options:
    max-size: "5m"       # 5MB
    max-file: "2"        # 保留 2 个文件
    compress: "true"
```

**生产环境**（日志量大，保留时间长）:
```yaml
logging:
  driver: "json-file"
  options:
    max-size: "50m"      # 50MB
    max-file: "10"       # 保留 10 个文件
    compress: "true"
```

**日志量极大**（使用 syslog 或其他驱动）:
```yaml
logging:
  driver: "syslog"
  options:
    syslog-address: "tcp://192.168.0.100:514"
    tag: "llm-gateway"
```

## 2️⃣ 应用日志文件轮转

### 日志文件

```
logs/
├── access.log        # HTTP 访问日志
├── app.log          # 完整应用日志
└── error.log        # 错误日志
```

这些日志文件**不会自动轮转**，需要手动配置。

---

### 方案 A：使用 logrotate（推荐）

#### 1. 创建配置文件

创建 `/etc/logrotate.d/llm-gateway`:

```bash
sudo nano /etc/logrotate.d/llm-gateway
```

#### 2. 添加配置

```
/export/llmuser/projects/llm-gateway-lite/logs/*.log {
    daily                       # 每天轮转
    rotate 30                   # 保留 30 天
    missingok                   # 如果日志文件不存在，不报错
    notifempty                  # 日志为空时不轮转
    compress                    # 压缩旧日志
    delaycompress               # 延迟一天压缩（保留昨天的未压缩）
    create 0644 root root       # 创建新文件的权限
    sharedscripts               # 所有日志共享脚本
    postrotate
        # 轮转后重载 nginx（不重启容器）
        docker exec llm-gateway-lite-llm-gateway-1 nginx -s reopen 2>/dev/null || true
    endscript
}
```

#### 3. 测试配置

```bash
# 测试配置语法
sudo logrotate -d /etc/logrotate.d/llm-gateway

# 强制执行一次轮转（测试）
sudo logrotate -f /etc/logrotate.d/llm-gateway

# 查看 logrotate 状态
cat /var/lib/logrotate/status | grep llm-gateway
```

#### 4. 验证效果

```bash
# 查看日志目录
ls -lh logs/

# 轮转后会生成
logs/access.log          # 当前日志
logs/access.log.1        # 昨天的日志（未压缩）
logs/access.log.2.gz     # 2 天前的日志（已压缩）
logs/app.log
logs/app.log.1
logs/app.log.2.gz
logs/error.log
logs/error.log.1
logs/error.log.2.gz
```

---

### 方案 B：使用脚本定期清理

#### 1. 创建清理脚本

创建 `scripts/rotate-logs.sh`:

```bash
#!/bin/bash
# 日志轮转脚本

LOGS_DIR="/export/llmuser/projects/llm-gateway-lite/logs"
KEEP_DAYS=30

cd "$LOGS_DIR" || exit 1

# 轮转日志文件
for log in access.log app.log error.log; do
    if [ -f "$log" ]; then
        # 生成带时间戳的文件名
        timestamp=$(date +%Y%m%d)
        
        # 复制并压缩
        cp "$log" "${log}.${timestamp}"
        gzip "${log}.${timestamp}"
        
        # 清空原文件
        > "$log"
    fi
done

# 删除旧日志
find "$LOGS_DIR" -name "*.log.*.gz" -mtime +${KEEP_DAYS} -delete

# 重载 nginx
docker exec llm-gateway-lite-llm-gateway-1 nginx -s reopen 2>/dev/null || true

echo "日志轮转完成: $(date)"
```

#### 2. 添加执行权限

```bash
chmod +x scripts/rotate-logs.sh
```

#### 3. 添加到 crontab

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点执行
0 2 * * * /export/llmuser/projects/llm-gateway-lite/scripts/rotate-logs.sh >> /var/log/llm-gateway-rotate.log 2>&1
```

### 方案 C：手动清理

```bash
# 查看日志文件大小
ls -lh logs/

# 压缩旧日志（保留最近 7 天）
find logs/ -name "*.log" -mtime +7 -exec gzip {} \;

# 删除 30 天前的压缩日志
find logs/ -name "*.log.gz" -mtime +30 -delete

# 清空当前日志（如果太大）
> logs/app.log
> logs/error.log
> logs/access.log

# 重载 nginx
docker-compose exec llm-gateway nginx -s reopen
```

## 📊 日志文件大小估算

### 预估规则

| 文件 | 请求量 | 每条日志大小 | 日增长 |
|------|--------|-------------|--------|
| **access.log** | 1000 req/day | ~200 字节 | ~200 KB |
| **app.log** | 1000 req/day | ~1 KB/请求（5-10 条） | ~1 MB |
| **error.log** | 50 失败/day | ~2 KB/次 | ~100 KB |

### 示例计算

**低流量场景**（100 req/天）:
- app.log: ~100 KB/天 × 30 天 = ~3 MB
- error.log: ~10 KB/天 × 30 天 = ~300 KB
- **总计**: ~3.5 MB/月

**中等流量**（10,000 req/天）:
- app.log: ~10 MB/天 × 30 天 = ~300 MB
- error.log: ~1 MB/天 × 30 天 = ~30 MB
- **总计**: ~350 MB/月

**高流量**（100,000 req/天）:
- app.log: ~100 MB/天 × 30 天 = ~3 GB
- error.log: ~10 MB/天 × 30 天 = ~300 MB
- **总计**: ~3.5 GB/月

### 压缩效果

使用 gzip 压缩后，日志文件通常可以压缩到 **10-20%** 的原始大小：
- 未压缩: 100 MB
- 压缩后: 10-20 MB

## 🎯 推荐配置

### 开发环境

**Docker 容器日志**:
```yaml
max-size: "10m"
max-file: "3"
```

**应用日志文件**: 手动清理或不配置轮转

---

### 生产环境（低流量 <1000 req/天）

**Docker 容器日志**:
```yaml
max-size: "10m"
max-file: "5"
compress: "true"
```

**应用日志文件**: logrotate
```
daily
rotate 30
compress
delaycompress
```

---

### 生产环境（中等流量 1000-10000 req/天）

**Docker 容器日志**:
```yaml
max-size: "20m"
max-file: "7"
compress: "true"
```

**应用日志文件**: logrotate
```
daily
rotate 14
compress
notifempty
```

---

### 生产环境（高流量 >10000 req/天）

**Docker 容器日志**:
```yaml
max-size: "50m"
max-file: "10"
compress: "true"
```

**应用日志文件**: logrotate
```
daily
rotate 7
compress
notifempty
size 100M  # 或者达到 100M 就轮转
```

或考虑使用外部日志系统（ELK、Loki 等）

## 🔍 监控日志大小

### 定期检查

```bash
# 查看日志目录总大小
du -sh logs/

# 查看各文件大小
du -h logs/* | sort -h

# 监控磁盘使用情况
df -h | grep -E "Filesystem|/$"
```

### 设置告警

当日志目录超过阈值时告警：

```bash
#!/bin/bash
# 检查日志大小并告警

LOGS_DIR="/export/llmuser/projects/llm-gateway-lite/logs"
THRESHOLD_MB=1000  # 1GB

current_size=$(du -sm "$LOGS_DIR" | awk '{print $1}')

if [ "$current_size" -gt "$THRESHOLD_MB" ]; then
    echo "⚠️  警告: 日志目录已达到 ${current_size}MB (阈值: ${THRESHOLD_MB}MB)"
    # 发送告警到 Slack/钉钉/邮件等
fi
```
